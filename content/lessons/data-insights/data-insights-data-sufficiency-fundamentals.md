## Overview

Data Sufficiency (part of Data Insights on the GMAT Focus Edition) doesn't ask you to *solve* —
it asks whether you *could*. You're given a question and two statements, and you decide which
statements provide enough information to answer it. The five answer choices are always the same,
so learn them cold.

## The five answer choices

- **A** — Statement (1) ALONE is sufficient, but statement (2) alone is not.
- **B** — Statement (2) ALONE is sufficient, but statement (1) alone is not.
- **C** — BOTH statements TOGETHER are sufficient, but NEITHER alone is.
- **D** — EACH statement ALONE is sufficient.
- **E** — The statements TOGETHER are still NOT sufficient.

## Core concepts

**"Sufficient" means a single, definite answer.** For a *value* question ("what is \(x\)?"), a
statement is sufficient only if it pins down exactly one value. For a *yes/no* question ("is
\(n\) even?"), a statement is sufficient if it always gives the same answer — a definite "yes" and
a definite "no" are *both* sufficient. An "it depends" is insufficient.

**Use the AD / BCE decision tree.** Test statement (1) first:

- If (1) is sufficient, the answer is **A or D**. (Now test (2): sufficient → D, not → A.)
- If (1) is not sufficient, the answer is **B, C, or E**. (Test (2): if sufficient → B; if not, test them together → C if enough, E if not.)

**Evaluate each statement independently first.** Don't let information from statement (1) leak
into your judgment of statement (2). Only combine them in the final "together" step.

## Worked example

*Is the integer \(n\) even?* (1) \(n^2\) is even. (2) \(3n\) is even.

- (1): odd² is odd, so \(n^2\) even forces \(n\) even → definite "yes" → **sufficient**.
- (2): 3 is odd, so \(3n\) even forces \(n\) even → definite "yes" → **sufficient**.

Each alone gives a definite answer → **D**.

## Common traps

- **The C-trap.** Combining the statements often *feels* safe, but if either works alone the
  answer is A, B, or D — and picking C loses the point. Always test each alone first.
- **Solving too far.** You don't need the actual value, only whether it's *determined*. Stop as
  soon as you know a unique answer exists.
- **Forgetting yes/no can be "definitely no."** A statement that guarantees "no" is sufficient.

## Key takeaways

- Memorize the five choices and the AD/BCE tree.
- "Sufficient" = yields one definite answer (value) or a consistent yes/no.
- Judge each statement alone before combining; beware the reflex to choose C.
